# AI-Accelerated Test Automation: Test Plan

**Scope:** UI and API testing for the Sandbox Application (Login & New Test Case features).
**Strategy:** Hybrid test approach (combining UI actions with API setup/teardown) to optimize execution speed.
**Total Test Cases:** 32

## 1. Authentication & Security (Login Feature)

### UI Flow - Positive
* **TC01:** Successfully log in with valid Email and Password; verify redirection to dashboard.
* **TC02:** Verify "Remember me" toggle maintains session state (cookie/local storage persistence).
* **TC03:** Verify "Forgot Password" link redirects to the correct page.
* **TC04:** Verify "Admin" link in the top menu redirects to the admin login page.
* **TC05:** Verify "Login" link in the top menu correctly routes to `/login` or remains on the current page.

### UI Flow - Negative & Validation
* **TC06:** Attempt login with unregistered email; verify clear error message.
* **TC07:** Attempt login with valid email but incorrect password; verify clear error message.
* **TC08:** Frontend Validation: Submit empty email field; verify no backend API call is made and error shows.
* **TC09:** Frontend Validation: Submit empty password field; verify no backend API call is made and error shows.
* **TC10:** Frontend Validation: Input invalid email format (e.g., `user@.com`); verify no backend API call is made.

### Security & Usability
* **TC11:** Security: Verify the login page loads strictly over HTTPS.
* **TC12:** Security: Verify the password input field is masked (`type="password"`).
* **TC13:** Usability: Verify labels and placeholders ("Email", "Password") are clearly visible.

### API Flow & Performance
* **TC14:** API Positive: Send valid credentials to the login endpoint; verify 200 OK and store the returned Bearer Token.
* **TC15:** API Negative: Send invalid credentials to the login endpoint; verify 401 Unauthorized.
* **TC16:** API Performance: Measure the login API endpoint response time; assert it resolves under 1000ms.

## 2. Test Management (New Test Case Feature)

### Hybrid Flow (API Setup for UI Test)
* **TC17:** Hybrid Setup: Inject the stored Bearer Token directly into the browser context to bypass the UI login screen, and navigate directly to the "New Test Case" page.

### UI Flow - Positive
* **TC18:** Successfully create a test case with all mandatory fields (Title, Expected Result, 1 Test Step).
* **TC19:** Successfully create a test case including the optional "Description" field.
* **TC20:** Add multiple test steps using the "Add Test Step" button; verify order is preserved.
* **TC21:** Toggle the "Automated" switch ON; submit and verify the test case is saved with automated status.

### UI Flow - Negative & Validation
* **TC22:** Attempt submission with an empty "Title"; verify error message.
* **TC23:** Attempt submission with an empty "Expected Result"; verify error message.
* **TC24:** Attempt submission with zero "Test Steps"; verify error message.

### Usability
* **TC25:** Verify pressing `Enter` in an input field does not submit the form prematurely if required fields are missing.
* **TC26:** Verify the "Back arrow" successfully navigates back to the Test Case List.
* **TC27:** Verify correct placeholders are displayed ("Title", "Description", "Expected Result", "Test step").

### API Flow & Data Cleanup
* **TC28:** API Positive: Create a new test case via direct API POST request using the Bearer Token; verify 201 Created.
* **TC29:** API Negative: Attempt to create a test case via API missing the "Title" payload; verify 400 Bad Request.
* **TC30:** Hybrid Teardown (Data Cleanup): At the end of the test suite, use the API to bulk-delete all test cases generated by TC18-TC21 and TC28 to prevent junk data buildup.

### Performance & Network Anomalies
* **TC31:** API Performance: Measure the test case creation API endpoint response time; assert it resolves under 1000ms.
* **TC32:** UI Network Analysis: During UI test case creation (TC18), monitor the browser network tab to ensure no duplicate API requests are fired and no static assets throw 404/500 errors.
